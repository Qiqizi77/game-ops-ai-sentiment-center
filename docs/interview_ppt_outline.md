# 面试讲解 PPT 大纲

## 1. 项目背景

- 游戏发行运营需要跨平台、跨版本、跨地区监控玩家反馈。
- V1 解决单游戏舆情看板，V2 升级为多 Agent 舆情中台。

## 2. 核心架构

- Collector、Analyzer、Alert、Reporter 四 Agent。
- SQLite 作为轻量消息总线和状态持久化。
- FastAPI 提供 API Gateway 与 OpenAPI 文档。

## 3. 算法亮点

- TF-IDF + Cosine Similarity + DBSCAN 风格聚类。
- 3σ + 移动基线异常检测。
- 细粒度情绪、意图识别、实体抽取、相关性判断。
- 版本发布前3天样本预测最终好评率。

## 4. 企业级能力

- 多游戏配置与数据隔离字段。
- API Key、角色、限流、审计日志。
- Agent 成功率、延迟、数据质量、告警和工单。

## 5. Varsapura 定制

- 开放世界性能、贴图、穿模、空气墙、AI NPC、昼夜天气。
- 全球发行地区隔离与多语言摘要。
- 自动工单、智能回复、官方话术建议、回应效果追踪。

## 6. 业务价值

- 将社区噪声聚类为可处理的问题。
- 将高危负面舆情从小时级发现压缩到分钟级。
- 让版本复盘从人工汇总升级为自动报告。

## 7. 可落地路线

- Demo：SQLite + mock adapter。
- 试点：接入飞书/JIRA，启用真实 API Key。
- 生产：采集节点拆分，数据库替换 PostgreSQL，接入企业观测平台。
